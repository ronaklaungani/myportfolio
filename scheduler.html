Weekly Schedule Viewer (Summary)

Role

UX & UI Designer

Platform(s)

Native Mobile App

Duration

1 week

Tools

Sketch, Zeplin

Methods

Conceptualization, Sitemap, Wireframing, Information Architecture, UI Design

Objective

Show wireframe and design for a UI element that enables users to select multiple ranges of time across the week so as to create a weekly schedule.

Overview

The UX of this product is inspired by many examples currently used by consumers and is a combination of the best elements from these examples; resulting in a very simple yet efficient product for the users.
The navigation, like that of iOS, is linear making it really simple to move around inside the app. The color scheme is also designed keeping the iOS guidelines in mind - with one key color suggesting interaction.

Interface Design

The Months View

1 - A month-year picker, right at the top - allowing the user to navigate to the desired month-year combination quickly.

2 - The “Today” button, to quickly navigate to the present day and access the tasks for the day.

3 - The calendar view is more informative in terms of number of tasks and events scheduled for the day. Clicking on any date tile takes the user to the detailed tasks view of the day in the “Week View” format.

4 - The “Add” floating action button is inspired by android’s material design, allowing user to quickly add a task/event - available on all primary the pages.

The Week View

4 - Back to the month view where the user can easily navigate to a different month/year.

6 - The page displays the tasks aligned for just the seven days of the week. By swiping left or right, the user can easily navigate to the next or the previous week’s task list.

7 - The add button here works exactly like the floating action button on the home page but it opens the “Add New” form with the date field pre-filled.

8 - The cards format, inspired by Android, is much more informative and has a different color to make it the most prominent content on the screen.

9 - The button here at the bottom of each day’s task list allows the user to enter the “day view” format for that particular date. It’s a more interactive UI compared to existing scheduling examples in the market.

The Day View





10 - Back to the weekly-view.

11 - The page displays the time-table for just one day, by swiping left or right, the user can easily navigate to the next or the previous day’s time-table.

12 - The tasks in the card format are now aligned with the time slots, giving the user a view of all the empty slots and the scheduled slots.

Easy Scheduling



13 - User can add details like the title/purpose of the task and details like notes, invitees, location, etc.

14 - The user can also attach a link which can be a reference for the task or an URL for a video call.

15 - The time scheduling, inspired by iOS is a really easy process. It allows the user to schedule hourly tasks and also all-day tasks.

Again. And Again.



Daily: User can schedule the task for every 1, 2, 3, etc. days. Clicking on the button “Every 1 day”, the user can edit this data.
Weekly: User can schedule the task to reoccur on a day/days of the future weeks.
Monthly: User can schedule the task to reoccur on the same date in every 1, 2, 3, etc. months.
Yearly: User can schedule the task to reoccur on the same date-month combination in every 1, 2, 3, etc. years.



The Task in Hand




16 - The task is of upmost importance here.

17 - User can edit the complete series of reoccurring schedules OR just one of these schedules from the series. User gets the flexibility to alter plans on weekly basis.

18 - User can also delete a scheduled task OR even mute an upcoming task reminder.
